# JourneyPlanner · AI 地图旅游规划 MVP

> **省心、安心的智能旅程规划。**
> 用户在**微信**里加上我们的**客服（云端机器人）**，把心仪的房间、景点、小红书/去哪儿笔记随手转发过去 → AI 帮你**汇总信息、读评论避雷、统筹每间房到各景点的距离** → 结果直接在会话里回（避雷图文 + 地图），帮你安心做决策。

**目标**：2026 年暑假前上线，获取首批几千名种子用户。

---

## 核心卖点与三大优势

**一句话**：把"自己开十几个 App、来回比价比位置、踩雷全靠运气"的累活，变成"转发给客服，AI 帮你整理好"。

| 优势 | 是什么 | 用户痛点 |
| --- | --- | --- |
| **① 精简找房 + 距离统筹** | 看到中意的房间直接转发给企微客服，客服记录后，系统统筹**每间候选房相对于所选几个景点的距离**，在地图上直观对比 | 选酒店时最难的不是房间本身，而是"它离我要去的几个景点到底方不方便" |
| **② 多源信息汇总** | 小红书、去哪儿、携程、美团……各种链接/笔记一起转发，**AI 整合成一份**结构化清单 | 信息散落在十几个平台，自己整理累到崩溃 |
| **③ AI 避雷壁垒** | AI 自动读取房源评论，按四维打分并总结**潜在雷点**，提前预警 | 评论几百条没人看得完，踩雷全凭运气 |

> ③ 是核心壁垒；①② 是让用户"省心"的体验，三者合起来才是"省心安心"。

## 核心功能

1. **转发即导入** —— 房间 / 景点 / 小红书 / 去哪儿等链接随手转发到企业微信客服。
2. **多源信息汇总** —— AI 把多平台内容整合成结构化清单（优势②）。
3. **AI 自动避雷** —— LLM 读评论，按「卫生 / 隔音 / 设施 / 服务」四维打分 + 雷点总结（优势③）。
4. **距离统筹 + 地图可视化** —— 统筹每间候选房到所选景点的距离；地图上展示地铁线路、景点 POI、酒店 POI（红/黄/绿 三色避雷等级）（优势①）。
5. **智能交通概览** —— 酒店↔景点 路径时间，一键拉起外部地图 App 导航。

## 开发阶段

项目分两阶段推进，**先打通链路，再做功能**。

### 阶段一：打通「微信客服 ↔ 云端」+ 实测携程小程序卡片 ← 当前阶段
目标：① 用户在微信加客服发消息 → 云端收到、解密、打印；② 转一张携程酒店卡片，把 `pagepath` 打印出来（决定后续解析能否成）。
详细 step-by-step 见 **[docs/SETUP-phase1.md](docs/SETUP-phase1.md)**。
> MVP 结果都在微信客服会话里回（避雷图文 + 高德静态地图图片），不做自有域名 H5，**0 备案**。

### 阶段二：核心功能（待展开）
多源链接解析（小红书 / 去哪儿 / 携程 / 美团，link & 小程序卡片）→ AI 信息整合 → LLM 评论避雷 → 入库 → 候选房↔景点 距离统筹 → H5 地图渲染 → 一键导航。

## 数据模型（来自计划书，阶段二落地）

- `users`：`id` / `wecom_user_id` / `created_at`
- `cities`：`id` / `name` / `adcode`(高德行政区划码)
- `hotels`：`id` / `user_id` / `city_id` / `name` / `source_url` / `latitude` / `longitude` / `rating` / `ai_rating_{hygiene,soundproof,facility,service}` / `ai_red_flags`(JSON) / `ai_summary` / `created_at`
- `attractions`：`id` / `city_id` / `name` / `longitude`(+纬度) / `type`
